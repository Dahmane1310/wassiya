# Mobile (Expo / React Native)

`apps/mobile` is Expo Router + Convex + WorkOS PKCE auth. A few things differ hard from the web apps.

## When to use
- Any change under `apps/mobile`, native modules, dev builds, mobile auth/token/SecureStore, Metro resolution.

## When NOT to use
- Web/admin (Next.js) or landing (Astro) — different runtimes and patterns.

## Hard requirements
- **A development build is mandatory — Expo Go will NOT work.** Auth uses a custom OAuth scheme callback and a native WebCrypto polyfill, neither of which Expo Go supports. Use `npx expo prebuild` + `npx expo run:android` (or `run:ios` on macOS).
- **The WebCrypto polyfill must be the first import.** Entry `index.ts` does `import "./src/polyfills"` before `expo-router/entry`. `src/polyfills.ts` calls `polyfillWebCrypto()` and shims `crypto.subtle.digest` via `expo-crypto` (WorkOS PKCE needs SHA-256). Putting any import before it breaks sign-in with "Cannot read property 'digest' of undefined".

## Auth & state
- WorkOS **PKCE**, secret-less: only the public client id ships (`EXPO_PUBLIC_WORKOS_CLIENT_ID`). **Never** put `WORKOS_API_KEY` in the bundle. Flow in `src/lib/auth.ts`.
- **Tokens are sensitive → `expo-secure-store`** under separate keys. Do NOT persist them via Zustand `persist`/AsyncStorage (a single JSON blob can exceed iOS's ~2KB/key, and it's unencrypted).
- Auth state lives in a Zustand store (`src/stores/auth.ts`) exposing a stable `fetchAccessToken` (refreshes via `authenticateWithRefreshToken`) for `ConvexProviderWithAuth`. No React `AuthProvider`.
- Client/UI state → Zustand (`src/stores/preferences.ts`, persisted to AsyncStorage). Server state stays in Convex.
- See `auth-wiring` for the backend/redirect-URI side (the mobile scheme must be a registered redirect URI).

## Metro / monorepo
- Expo auto-configures Metro for the pnpm workspace. `@workspace/backend` exposes its API only through its `exports` field, so `metro.config.js` sets `config.resolver.unstable_enablePackageExports = true` so `import { api } from "@workspace/backend/api"` resolves.
- Native dirs (`android/`, `ios/`, `.expo/`) are generated by `expo prebuild` and **gitignored** (CNG) — regenerate, don't commit. They live where the app is, so they don't survive a folder rename — re-run prebuild.

## Gotchas
- Pin `@workos-inc/node` to the `@convex-dev/workos-authkit` component's supported peer range (`^9`) so the workspace resolves one version.
- `atob` is a Hermes global (RN ≥ 0.74) used for JWT `exp` decoding; if a future RN lacks it, swap to a manual base64url decoder.
- **Windows file locks:** a running `turbo dev`/Metro locks `apps/mobile` dirs — stop it before renaming/moving the app folder.

## Verify
- Dev build launches; sign-in completes the WorkOS browser flow and returns via the scheme; a Convex `currentUser` query shows a non-null identity; relaunch stays signed in (SecureStore hydration).
- `pnpm --filter mobile typecheck` passes.
